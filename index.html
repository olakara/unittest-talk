Unit Testing

  • A unit test is a code written to test a small piece of functionality.

    • Tests ensure:

    • The code meets expecations and specifications
    • The code continues to meet expectation over time

Types of software testing

Anotomy of a Unit Test

  • Setup
  • Prepare the appropriate input
  • Call the method to test
  • Check the result
  • Tear down

Why JavaScript should be tested?

  • JavaScript has become first class citizen in our projects
  • JavaScript has moved from client side to server side
  • Prodution code should be tested!

Quick review of unit testing

  • Use unit tests to drive development and design
  • Write the test first and then the code
  • Test coverage of your code remains high
  • The test suite is tipically run frequently

Jasmine

  • Behavior-driven development framework for testing JavaScript
  • Its a standalone framework
  • Does not require DOM
  • Jasmine tests itself across many browsers (Safari, Chrome, Firefox, PhantomJS, and new Internet Explorer) as well as Node, Ruby & Python

Installing Jasmine - Standalone

  • Download the standalone distribution
  • Create a Jasmine directory in your project :
    λ mkdir my-project/jasmine
  • Move the distribution to the jasmine folder & unzip
  • Create the main HTML file with the necessary files:

Installing Jasmine - NPM 


λ npm install jasmine-core --save-dev
  • Install along with any test runner..
  • You can use jasmine for testing Node,Ruby or Python

Jasmine Specs 




describe("A suite", function() {
  it("contains spec with an expectation", function() {
    expect(true).toBe(true);
  });
});

Expectations

Expectations are built with the function expect which takes a value, called the actual. It is chained with a Matcher function, which takes the expected valu.
Example: 

expect(true).toBe(true);

Matchers

  • Matchers are functions that implements the comparison between the actual and the expected value
  • It is responsible for reporting to Jasmine if the expectation is true or false
  • You can create your own matchers in Jasmine
  • Examples: 
    
*.toBe(*)
*.not.toBe(*)
*.toEqual(*)
*.toBeDefined()
*.toBeUndefined()
*.toBeNull()
*.toContain(*)
*.toBeLessThan(*)
*.toBeTruthy(*)

Spies

A person employed to secretly obtain information on an enemy!!
  • With Jasmine you can sky on any function and track calls to it and all arguments
  • A spy only exists in the describe or it block in which it is defined
  • Can be used when you deal with 3rd party libraries in your application
  • Simple Example (Creation): 
    
spyOn(person, 'walk');
createSpy('run');

Building the ultimate spy!

What if you need to spy on multiple functions or mock a whole object?

Solution: 


createSpyObj(person, ['walk', 'talk', 'eat', 'run']);
Returns an object that has a property for each string you passed in!

Matchers for the Spies 


*.toHaveBeenCalled()
*.toHaveBeenCalledWith(*)
*.callThrough()           -> spyOn(shape, 'getShape').and.callThrough()
*.returnValue()           -> spyOn(shape, "getShape").and.returnValue('Circle');
*.callFake(function() {})

Karma

  • Karma is a JavaScript test runner aimed at bringing productive testing environment to developers
  • It was formaly known as testacular
  • It runs test on real browers
  • Can integrate with most of the test frameworks available
  • Easy to integrate with CI

Installing Karma

Install Karma locally in the project's directory. 


λ npm install karma --save-dev
Install plugins: 

λ npm install karma-jasmine karma-chrome-launcher --save-dev
Setup the shortcut! 

λ npm install -g karma-cli
Note: You will have to load other unit test frameworks if necessary

Karma Configuration

  • Initial configuration is created by: 
    
λ karma init
  • The Karma configuration file can be written in JavaScript or CoffeeScript and is loaded as a regular Node.js module.
  • Default configuration files the karma CLI looks for:
    1. ./karma.conf.js
    2. ./karma.conf.coffee
    3. ./.config/karma.conf.js
    4. ./.config/karma.conf.coffee
